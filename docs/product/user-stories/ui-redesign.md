# UI 改版 — User Stories

> Epic: UI-REDESIGN | 参考: flowkey.com 设计风格

---

### US-UI-01 高品质视觉体验

**背景：** 当前界面大量使用 emoji 作为图标，配色单一（金色过度使用），整体观感偏简陋，缺乏专业钢琴学习 App 的品质感。用户在首次打开 App 时需要感受到专业、优雅、值得信赖的氛围。

**用户故事：** 作为钢琴学习者，我希望 App 看起来精致专业，让我有信心在这里投入时间学习。

**AC：**
- Given 用户首次打开 App
- When 浏览各个页面时
- Then 所有图标使用统一的矢量图标（非 emoji），配色方案优雅克制，整体视觉统一且有高级感

---

### US-UI-02 沉浸式深色主题

**背景：** 钢琴练习场景通常在较暗的环境中进行，深色主题更护眼。当前深色主题层次不够分明，缺乏空间感和呼吸感。

**用户故事：** 作为经常在晚上练琴的用户，我希望 App 有舒适的深色界面，不刺眼且层次分明。

**AC：**
- Given 用户在任何页面
- When 浏览内容时
- Then 背景、卡片、弹窗有明确的层次区分，重要信息突出，次要信息退隐，大量留白让界面不拥挤

---

### US-UI-03 清晰的首页信息架构

**背景：** 首页是用户最常访问的页面，需要快速了解自己的学习状态和下一步行动。当前首页信息堆叠较多，缺乏焦点。

**用户故事：** 作为每天打开 App 的用户，我希望首页能让我一眼看到今日目标、当前课程进度，并快速开始练习。

**AC：**
- Given 用户进入首页
- When 查看页面内容时
- Then 页面顶部有简洁的欢迎语和练习数据，中部突出当前课程进度卡片，底部展示推荐内容，整体信息层次清晰
- Given 用户想快速开始练习
- When 点击最醒目的行动入口
- Then 直接进入上次未完成的课程或练习

---

### US-UI-04 优雅的曲库浏览体验

**背景：** 曲库是用户发现新内容的核心入口。当前列表式排布比较单调，缺少封面图等视觉吸引力。

**用户故事：** 作为想找曲子练习的用户，我希望曲库页面有吸引力的曲目展示，能快速按难度和风格找到我想练的曲子。

**AC：**
- Given 用户进入曲库页面
- When 浏览曲目列表时
- Then 每首曲目有精美的封面卡片展示，包含曲名、作者、难度、时长等关键信息
- Given 用户想筛选特定难度的曲目
- When 点击难度筛选
- Then 列表平滑过渡展示对应难度的曲目

---

### US-UI-05 激励性的练习数据呈现

**背景：** 练习统计能帮助用户感受到自己的进步，是持续学习的重要动力来源。当前练习页的数据展示缺乏视觉冲击力。

**用户故事：** 作为坚持练琴的用户，我希望看到自己的练习成果以直观漂亮的方式呈现，让我有成就感。

**AC：**
- Given 用户进入练习统计页
- When 查看练习数据时
- Then 关键数据（总时长、连续天数、完成曲目数）以大字体突出展示，练习趋势以优雅的图表呈现
- Given 用户查看某次练习记录
- When 查看详情时
- Then 展示分数、准确率、评级等信息，视觉表现有成就感

---

### US-UI-06 专业的个人中心

**背景：** 个人中心是用户了解自身等级、成就、订阅状态的汇总页面。当前设计比较扁平，缺乏个性化感觉。

**用户故事：** 作为长期使用的用户，我希望个人中心体现我的成长轨迹，包括等级、成就徽章等，有专属感。

**AC：**
- Given 用户进入个人中心
- When 查看页面时
- Then 头像区域有精致的视觉设计，等级和经验值以进度条清晰展示，成就徽章以网格形式有序排列
- Given 用户已获得某些成就
- When 查看成就区域时
- Then 已解锁的成就有高亮展示，未解锁的灰度显示但可预览条件

---

### US-UI-07 流畅的页面转场与微交互

**背景：** 动画和过渡效果能提升操作的流畅感和品质感，让用户操作更有反馈。当前 App 几乎没有动画效果。

**用户故事：** 作为日常使用 App 的用户，我希望页面切换和操作交互都有流畅自然的动画效果，操作起来有质感。

**AC：**
- Given 用户在 App 中导航
- When 切换页面、点击按钮、展开卡片时
- Then 有平滑的过渡动画，按钮点击有触感反馈，列表滚动流畅
- Given 数据加载中
- When 等待内容出现时
- Then 有优雅的骨架屏或加载动画，而非空白等待
