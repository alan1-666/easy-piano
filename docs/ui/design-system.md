## 1. 色彩系统

### 1.1 主色板

| 用途 | 色值 | 色样 | 说明 |
|------|------|------|------|
| 深色背景 | `#1A1A2E` | ████ | App 主背景色 |
| 次级背景 | `#16213E` | ████ | 卡片、弹窗背景 |
| 表面色 | `#0F3460` | ████ | 输入框、分割线背景 |
| 强调色/金色 | `#FFD700` | ████ | 判定线、高亮、主要按钮 |
| 白色文字 | `#FFFFFF` | ████ | 主文字色 |
| 次级文字 | `#A0A0B0` | ████ | 辅助说明文字 |
| 禁用文字 | `#555570` | ████ | 不可点击状态 |

### 1.2 语义色

| 用途 | 色值 | 说明 |
|------|------|------|
| 左手音符 | `#4A90D9` | 蓝色系，左手演奏标识 |
| 右手音符 | `#50C878` | 绿色系，右手演奏标识 |
| Perfect 评价 | `#FFD700` | 金色，最高评价 |
| Great 评价 | `#4A90D9` | 蓝色 |
| Good 评价 | `#50C878` | 绿色 |
| Miss 评价 | `#E74C3C` | 红色，失误标识 |
| 成功/已连接 | `#2ECC71` | 绿色，状态正常 |
| 警告 | `#F39C12` | 橙色 |

### 1.3 渐变

```
主按钮渐变:     #FFD700 → #FFA500  (从左到右)
经验值条渐变:   #4A90D9 → #50C878  (从左到右)
判定线光晕:     #FFD700 40% → #FFD700 0%  (径向渐变)
背景氛围渐变:   #1A1A2E → #16213E  (从上到下)
```

## 3. 间距系统

### 3.1 基础网格

采用 **4px** 为基础网格单位，所有间距应为 4 的倍数。

```
4px   — 最小间距 (图标与文字间距)
8px   — 紧凑间距 (列表项内部)
12px  — 小间距 (相关元素之间)
16px  — 标准间距 (卡片内边距、段落间距)
24px  — 大间距 (模块之间)
32px  — 区块间距 (页面大区块之间)
48px  — 页面边距 (底部安全区)
```

### 3.2 标准页面边距

```
┌────────────────────────────────┐
│←16px→┌──────────────────┐←16px→│  iPhone (390px宽)
│      │   内容区域         │      │
│      └──────────────────┘      │
└────────────────────────────────┘

┌──────────────────────────────────────────┐
│←24px→┌──────────────────────────┐←24px→│  iPad (1024px宽)
│      │      内容区域              │      │
│      └──────────────────────────┘      │
└──────────────────────────────────────────┘
```

## 6. 动画规范

| 动画类型 | 时长 | 缓动函数 | 使用场景 |
|---------|------|---------|---------|
| 页面切换 | 300ms | ease-in-out | 路由导航 |
| 弹窗弹入 | 250ms | spring(1, 0.8, 0) | 模态弹窗 |
| 按钮按下 | 100ms | linear | 按钮反馈 |
| 卡片展开 | 200ms | ease-out | 列表展开 |
| 评价弹出 | 150ms | spring | Perfect/Great 等弹出 |
| 评价消失 | 500ms | ease-in | 评价文字淡出 |
| 音符下落 | 持续 | linear | 游戏核心动画 (与BPM同步) |
| 星星揭晓 | 400ms/颗 | spring(1, 0.5, 0) | 结算星级动画 |
| 进度条填充 | 600ms | ease-out | XP/进度增长 |

## 7. iPadOS 适配规则

### 7.1 布局策略

```
iPhone (390×844pt, 竖屏):
┌──────────────┐
│   单列布局    │
│   全屏游戏    │
│   底部Tab    │
└──────────────┘

iPad (1024×1366pt, 竖屏):
┌──────────────────────────┐
│       双列/多列布局        │
│    卡片网格 (2-3列)       │
│    游戏: 更宽键盘          │
│       底部Tab             │
└──────────────────────────┘

iPad (1366×1024pt, 横屏):
┌──────────────────────────────────┐
│  侧边栏  │     内容区域            │
│  导航    │     游戏: 全宽键盘      │
└──────────────────────────────────┘
```

### 7.2 关键适配规则

| 规则 | 说明 |
|------|------|
| 最小触摸区域 | 所有可点击元素 ≥ 44pt × 44pt |
| Split View 支持 | 支持 1/3、1/2、2/3 分屏 |
| 键盘缩放 | 游戏虚拟键盘宽度 = 屏幕宽度，键宽按比例缩放 |
| 字号不缩放 | iPad 上字号与 iPhone 一致，通过增加列数利用空间 |
| 安全区适配 | 所有页面遵守 `SafeAreaView` 约束 |
| 游戏横屏 | iPad 游戏页面默认横屏，充分利用宽度 |

### 7.3 断点定义

| 断点 | 宽度范围 | 典型设备 | 列数 |
|------|---------|---------|------|
| compact | < 428pt | iPhone (竖屏) | 1列 |
| regular | 428-834pt | iPad (1/3分屏)、iPhone Max横屏 | 2列 |
| wide | > 834pt | iPad (全屏/横屏) | 3列 |

## 9. 暗色模式

EasyPiano 默认采用暗色主题（深色背景），与钢琴/游戏的沉浸感匹配。暂不提供浅色模式切换。

所有色值在上述色板基础上设计，确保:

| 对比度要求 | 标准 |
|-----------|------|
| 正文文字 vs 背景 | ≥ 4.5:1 (WCAG AA) |
| 大标题 vs 背景 | ≥ 3:1 (WCAG AA) |
| 图标 vs 背景 | ≥ 3:1 |
| 可交互元素 | ≥ 3:1 + 非纯色彩区分 |
