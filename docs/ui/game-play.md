# 游戏主屏

> 核心游戏页面 — 下落音符弹奏界面  
> 版本: v1.0.0 | 更新日期: 2026-04-18  
> 这是 App 中最重要的页面，所有交互设计需精确到像素级别

---

## 1. 页面概述

全屏沉浸式游戏界面。音符从上方下落，玩家在判定线位置弹奏电钢琴上对应的键，获得实时评分和连击奖励。支持竖屏 (iPhone) 和横屏 (iPad) 两种布局。

## 4. 核心元素详细规格

### 4.1 音符方块

```
右手音符:                    左手音符:
┌──────────┐                ┌──────────┐
│          │                │░░░░░░░░░░│
│   绿色   │                │  蓝色    │
│  #50C878 │                │ #4A90D9  │
│          │                │░░░░░░░░░░│
│          │                │          │
└──────────┘                └──────────┘

黑键音符 (略深):
  右手黑键: #3DA863          左手黑键: #3A73B5

音符尺寸规则:
  宽度 = 对应琴键宽度 - 2px (左右各留1px间距)
  高度计算:
    全音符 (4拍):  speed × 4 × beatHeight
    二分音符 (2拍): speed × 2 × beatHeight
    四分音符 (1拍): speed × 1 × beatHeight
    八分音符 (0.5拍): speed × 0.5 × beatHeight
    
  beatHeight = 下落区域高度 / 可见拍数
  最小高度: 16pt (确保可见)

音符圆角: 4px
音符边框: 1px rgba(255,255,255,0.1) (微光泽)

弹过的音符:
  透明度从 1.0 → 0.0 (300ms 淡出)
  颜色变为对应判定颜色 (Perfect=金色, Miss=红色)
```

## 5. 虚拟键盘详细规格

### 5.1 键盘布局

```
可见音域: 根据歌曲自动计算，通常显示 1-2 个八度 (iPhone) 或 3-4 个八度 (iPad)

白键:
  宽度 (iPhone): 屏幕宽度 / 可见白键数
  宽度 (iPad): 屏幕宽度 / 可见白键数
  高度: 键盘区域全高
  背景: #F5F5F5 (微灰白)
  边框: 1px #D0D0D0
  圆角: 0 (底部) / 4px (顶部)

黑键:
  宽度: 白键宽度 × 0.65
  高度: 键盘区域 × 0.6
  背景: #2C2C2E
  位于相邻白键之间

音名标签:
  位于白键底部
  字号: 10pt Medium
  颜色: #888888
  只显示 C 音名 (C3, C4, C5...)
```

### 5.2 键盘高亮

```
MIDI 按下时:
  ┌──────┐
  │      │  对应键颜色变为:
  │  高  │    右手区域: #50C878 (绿色)
  │  亮  │    左手区域: #4A90D9 (蓝色)
  │      │  渐变: 从底部向上渐变
  └──────┘  过渡: 50ms 淡入

MIDI 松开时:
  高亮色 → 原色 (100ms 淡出)

判定颜色叠加:
  Perfect: 键上闪现金色波纹
  Miss: 键上闪现红色波纹
```

    │

## 12. 边界情况

| 场景 | 处理 |
|------|------|
| MIDI 断开 (游戏中) | 自动暂停 + 显示重连提示 |
| 来电/通知打断 | 自动暂停，返回后显示暂停面板 |
| App 进入后台 | 自动暂停，保存当前进度 |
| 屏幕旋转 (iPad) | 支持旋转，布局自适应，不中断游戏 |
| 极长音符 | 音符高度上限 = 下落区域高度，超出部分裁剪 |
| 极密集音符 | 确保最小间距 2px，避免视觉粘连 |
| 速度调整 | 暂停状态下调整，恢复后生效 |
| 电池低电量 | 降低光晕特效，优先保证帧率 |
